Input Test

This plot shows the running time (in seconds) of several input types to a program that performs no operations in neither the Mappers nor the Reducers. Hence it is a measure of the efficiency in setting up a job using different input formats. The input types considered were HipiImageBundles, SequenceFiles, the HARFileSystem, and many small image files: Note that specifying 10,000 small files or more to the system results in a setup phase that takes too long to complete without timing out.

Average Image with Culling Test

HIPI's CullMapper is a useful extension of the Hadoop Mapper class that allows a user to cull images that do not pass a certain set of criteria. In this experiment, the average image of the Canon PowerShot S500 digital camera is computed when the image taken had a resolution of 2592x1944. The relevant portion of the code responsible for performing the culling is copied below: 
    public static class MyMapper extends CullMapper<ImageHeader, FloatImage, NullWritable, FloatImage>
    {       
        public boolean cull(ImageHeader key) throws IOException, InterruptedException {
            if(key.getEXIFInformation("Model").equals("Canon PowerShot S500") && key.width == 2592 && key.height == 1944)
                return false;
            else
                return true;
        }
